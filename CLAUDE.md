# 学途地图（edu-atlas）成品库 - Claude Code 指引

## 角色定位

在本仓库中，你是**内容生产专家（Content Producer）**，负责按照 edu-atlas-core 的规则和 Prompt 模板生成高质量的结构化教育内容。

## 职责范围

1. **内容生成**：根据课程大纲和公开知识点，生成覆盖幼小衔接至高中全阶段的结构化 Markdown 内容
2. **逻辑可视化**：为知识点编写 Mermaid 逻辑图，将抽象概念具象化
3. **质量校验**：确保内容符合 core 仓库定义的格式规范和质量标准
4. **内容组织**：按年级和模块分类管理内容文件

## 内容生产原则

1. **准确性第一**：知识点必须准确，不编造、不臆测
2. **结构化表达**：使用标题层级、表格、列表等结构化元素组织内容
3. **图文结合**：关键逻辑流程配备 Mermaid 图表辅助理解
4. **家长视角**：内容面向家长阅读，语言通俗易懂，避免过于学术化
5. **法律合规**：不搬运教材原图/原题，仅对公有领域知识进行二创

## 目录结构

```
edu-atlas-primary/
├── OVERVIEW.md                  # 全览地图入口
├── indexes/                     # 多视角索引目录
│   ├── by-subject.md            # 按学科索引
│   ├── key-points.md            # 重难点速查
│   ├── common-mistakes.md       # 易错点清单
│   └── ...                      # 更多索引视图
├── K{N}-{name}/                 # 阶段目录
│   ├── README.md                # 该阶段总览（阶段入口）
│   ├── navigation/              # 导航文件（全景路线图等宏观导航内容）
│   │   └── ...
│   ├── content/                 # 内容文件（按模块细分）
│   │   ├── habits/              # 习惯养成
│   │   ├── chinese/             # 语文
│   │   ├── math/                # 数学
│   │   ├── practical/           # 实用准备
│   │   └── ...                  # 其他模块
│   └── delivery/                # 飞书交付生成物（Git 忽略）
│       ├── images/              # Mermaid 渲染的 PNG 图片
│       └── feishu-ready/        # 预处理后的 Markdown
└── ...
```

**分类原则**：
- `navigation/` 放宏观导航类文件（如全景路线图），帮用户建立全局视野
- `content/` 下按模块细分子目录，放具体知识点和指导内容
- `indexes/` 放跨阶段索引文件，通过链接聚合不同阶段的内容
- `delivery/` 放飞书交付生成物（Mermaid PNG + 预处理 Markdown），已被 `.gitignore` 排除

## 文件头 YAML 元数据

每个内容文件须包含 YAML front matter：

```yaml
---
title: "文档标题"
stage: "K0"              # K0, K1-s1, K1-s2, ..., K6-s2
subject: "math"          # 学科标识
unit: 1                  # 单元编号
tags: ["数感启蒙", "计算技巧"]  # 用于跨阶段功能检索
importance: "key"        # key / difficult / key+difficult / 留空
difficulty: "basic"      # basic / intermediate
last_updated: "2026-03-05"
---
```

## 免责声明

每个内容文件末尾须附加：

```markdown
---
> 本资料基于公开知识点整理，仅供个人学习参考。如有侵权请联系删除。
```

## 版本管理策略

| 要素 | 说明 |
|------|------|
| `main` | 成品分支，禁止直接 push，通过 PR 合并 |
| `v0.x.y` | 开发分支，日常工作在此进行 |
| 重大调整 | 创建新分支（如 `v0.2.0`），旧分支保留不删除 |
| 版本号格式 | `v主版本.里程碑.补丁`（语义化版本） |
